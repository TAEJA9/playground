Trend 1·2
채널톡운영 에이전트 COSBigQuery동적 SQLGemini
◤ As-Is · 기존 한계

전국의 수많은 중소상공인(SMB) 사장님들은 정작 ‘우리 고객이 누구인지, 지금 무엇을 원하는지’를 데이터로 알지 못한 채 감(感)에 의존해 결정을 내립니다. 주문·상담·결제 기록이 쇼핑몰·메신저·전화 등 서로 다른 시스템에 흩어져 있어, 한데 모아 보려면 전문 분석가나 개발자가 필요한데 중소기업엔 그럴 인력도 비용도 없습니다. 결국 ‘데이터는 쌓이는데 아무도 읽지 못하는’ 상태가 됩니다.

◤ To-Be · 혁신 아키텍처
  • 상담 채널(라이브챗·카톡·인스타·메일·라인·전화)을 채널톡 하나로 통합, 기간계 연동으로 상담의 80%를 AI 자동화(주문조회·반품 등)
  • 운영 에이전트 ‘COS(비서실장)’: 자연어 질문 → 연동 시스템 스스로 스터디 → 데이터 플래닝 → 동적 SQL 생성·분석 (데이터팀 없이 전 분석 수행)
  • 분석에 그치지 않고 행동까지: 이탈 고객 리스트 추출 → AI가 직접 전화로 사유 인터뷰·리텐션·미팅 예약·제안서 발송
  • 구글 클라우드 BigQuery 데이터웨어하우스 + Gemini(다국어·안정성·가성비). “모델은 원유, 그 위 하네스(컨텍스트)가 고부가가치”
💬 실제로 이렇게

현장 데모 — 대표가 채팅창에 “우리 회사 매출 얼마야?”라고 치자 즉시 분석. 이어 “최근 2주 이탈한 고객사 리스트”를 뽑고, 그 고객에게 AI가 직접 전화를 걸어 “기능은 만족했는데 가격이 좀…” 같은 실제 통화를 이어가며 사유를 듣고 미팅까지 제안했습니다.

🎯 기대효과 — 전담 데이터팀 없이도 실시간으로 의사결정하고, 떠난 고객을 AI가 자동으로 되살려 매출 회복까지 이어집니다.
▶ AX Idea

‘분석 → 행동’을 한 흐름으로 잇는다. 데이터를 BigQuery에 정제 적재하고 자연어→동적 SQL 분석 에이전트를 만든 뒤, 그 결과에 따라 실제 액션(알림·연락·티켓 생성)을 실행하는 후속 실행 에이전트를 짝으로 붙인다. 성능은 데이터 정제 수준이 좌우하므로 소스 정제를 선행한다.

◤ To-Be · 혁신 아키텍처
  • 가장 어려운 일이 아니라 ‘시간을 가장 많이 쓰는 일’부터 — 워크플로우 관찰로 타깃 선정
  • Gemini Enterprise 폐쇄 보안 환경으로 유출 리스크 해결
  • 투트랙: CTD 문서 효율화 + 제제·분석 데이터 활용 고도화
  • 위험평가 근거 초안 자동생성, 공정 비교표 산출, 공정 타당성 근거 초안, 과거 실패 데이터 재해석까지
💬 실제로 이렇게

강화된 규제에 맞춰 기존·신규 연구 자료를 넣으면 ‘제조 공정 위험평가 근거 초안’을 자동 생성. 여러 형식의 공정 문서를 넣으면 ‘공정 비교표’를 산출하고, 과거 실패한 실험 데이터까지 다시 넣어 신제품 아이디어로 재해석합니다.

🎯 기대효과 — 고급 연구인력이 서류 작업에서 벗어나 본질적 연구에 집중하고, 개인차 없이 문서 품질이 표준화됩니다.
▶ AX Idea

자동화 1순위는 ‘가장 어렵고 화려한 것’이 아니라 전문 인력이 시간을 가장 많이 쓰는 정형 문서 업무다. 서식이 표준화된 영역(보고서·규제문서·리뷰)을 먼저 잡고, 보안이 걸리면 Enterprise 폐쇄 환경으로 푼다. “AI를 위해 연구를 바꾸지 말고, 더 좋은 연구를 위해 AI를 쓴다.”

◤ As-Is · 기존 한계

건강관리 앱은 설치 한 달 뒤에도 계속 쓰는 사람이 5%도 안 됩니다(대부분 금방 떠납니다). 이유는 단순합니다 — 사용자가 먹은 것·운동·수면을 매번 직접 ‘기록’해야 하는데, 이게 귀찮아서 안 하게 됩니다. 그러면 앱은 그 사람에 대해 아무것도 모르는 상태가 되고, 애써 준비한 맞춤 건강 조언이 전부 쓸모없어집니다. ‘기록해야 이해하는데, 정작 기록을 안 한다’는 악순환입니다.

◤ To-Be · 혁신 아키텍처
  • 기록 허들 제거: CGM 혈당 자동측정, 호흡 소리로 수면 측정, 반지형 혈압, 비전 AI 식사 사진 인식, 라이브 OCR 공산품 등록
  • 먼저 말 거는 에이전트: 기록 감소·평소와 다른 행동을 감지하면 선제 너지(“어제 수면 5시간 → 오늘은 회복에 집중”)
  • 식사·운동·수면·혈당 데이터를 MCP로 연결해 하나의 맥락으로 통합 — 요청이 없을 때도 상호작용
💬 실제로 이렇게

“오늘 운동해라”가 아니라 “어제 수면이 5시간으로 짧았으니, 오늘은 강도 높은 운동보다 회복에 집중하자”처럼 먼저 말을 겁니다. 식사는 사진만 찍으면 비전 AI가 무엇을 얼마나 먹었는지 인식하고, 제품 뒷면을 비추면 라이브 OCR이 영양정보를 자동 등록합니다.

🎯 기대효과 — 기록 부담을 없애 이탈을 막고, 먼저 챙기는 케어로 지속 사용과 실제 건강 개선을 끌어냅니다.
▶ AX Idea

사용자가 요청하기 전에 먼저 개입하는 프로액티브 설계를 도입한다. 분산된 데이터 소스를 MCP로 묶어 에이전트가 맥락을 통합 이해하게 하고, 방해가 되지 않도록 너지 트리거(이상 징후 감지) 기준을 명확히 정의한다.

핵심 시사점 & AX Idea (실행 체크리스트)

11개 세션을 관통하는 4개 축과, 이번 분기부터 바로 실행할 단계별 AX Idea입니다.

A · 기술

AI를 사내 데이터에 어떻게 연결할까

여기어때 · 컬리 · 파스타 · 대원제약

AI 모델 자체보다, AI를 회사 데이터에 붙이는 ‘연결 방식’이 더 중요합니다.

▶ 실행 포인트
  1. MCP·ADK 표준화 — 슬랙·DB·메신저 같은 여러 사내 시스템을 하나하나 개별로 연결하지 말고, 공통 규격(MCP)으로 통일해서 연결한다. (여기어때)
  2. 지식 축적 — 대화가 끝나면 핵심을 사용자별로 저장해 다음 대화에서 회상하고, 반복적으로 유효한 판단 기준·선호는 장기 메모리에서 관리한다. (여기어때)
  3. 지식 유형별로 저장 방법을 다르게 — 코드는 코드대로, ‘왜 바뀌었는지’ 같은 이력은 요약해서, 문서는 기존 검색 방식으로. (컬리)
  4. 출처 강제 — ‘출처 없으면 답변 못 하게’를 AI에게 프롬프트로 부탁하는 게 아니라, 아예 시스템(코드)으로 막는다. (컬리)
  5. 민감한 데이터는 폐쇄망 — 보안이 중요한 데이터는 외부로 나가지 않는 엔터프라이즈 환경에서 처리한다. (대원제약)
B · 서비스

AI 할루시네이션을 어떻게 관리할까

위버스 · CJ올리브영

AI 헛소리(환각)는 완전히 없앨 수 없으니, 상황별로 다른 안전장치를 겹겹이 두는 것이 좋습니다.

▶ 실행 포인트
  1. 정확성이 중요한 업무(결제·환불) — AI가 알아서 판단하게 두지 말고, 미리 정해진 규칙(if-then 같은 분기)으로 처리한다. (위버스)
  2. 단순 지식 질문 — 검색 기반(RAG)으로 처리하되, FAQ를 먼저 찾고 없으면 지식베이스로 넘어가는 2단계 검색을 적용한다. (위버스)
  3. 최종 답변 전 크로스체크 — AI가 만든 답을 또 다른 AI가 “이거 사실 맞아?” 채점하고, 점수가 낮으면 사람이 검토·승인한다. (위버스)
  4. 스스로 똑똑해지는 시스템 — 사람들이 쓸수록 대화 데이터가 쌓이고, 자동으로 분류돼 FAQ가 계속 늘어나게 만든다(사람 승인 필요). (위버스)
C · 조직

AI를 깔아주는 게 끝이 아니라, 문화가 바뀌어야 한다

카카오뱅크 · CJ올리브영

좋은 AI 도구를 줘도 직원이 안 쓰면 의미가 없습니다. ‘기술 도입’이 아니라 ‘조직 문화·환경 만들기’가 필요합니다.

▶ 실행 포인트
  1. 앰버서더 제도 — 각 부서에서 AI를 잘 쓰는 사람을 뽑아 전파자 역할을 맡긴다. (카카오뱅크)
  2. 에이전트/프롬프트 뱅크 — 사내 해커톤 등에서 나온 좋은 결과물(잘 만든 에이전트)을 모아 공유·재사용한다(구글 에이전트 공유 기능). (카카오뱅크)
  3. 암묵지 공유 — 개인 머릿속에만 있던 노하우를 공유 드라이브에 모으는 것이, 화려한 에이전트를 만드는 것보다 먼저다. (CJ올리브영)
  4. 지표를 ‘도입률’이 아니라 ‘지속 사용률’로 — 한 번 써보고 마는 게 아니라, 한 달 뒤에도 계속 쓰고 있는지를 성공 기준으로 삼는다. (CJ올리브영)
D · 신뢰·개인

책임 있는 AI + 개인의 역할 변화

카카오 Responsible AI · 루시 · 파주

AI가 발전할수록 신뢰 문제가 중요해지고, 일반 구성원도 직접 만들어 쓰는 사람으로 바뀌는 변화가 필요합니다.

▶ 실행 포인트
  1. 워터마킹·출처 표시 — AI가 만든 콘텐츠라는 걸 처음부터 표시하고, 나중에 편집됐는지도 자체 검증한다. (카카오)
  2. ‘딸깍 결과’ 경계 — AI가 한 번에 뱉어준 결과를 그대로 믿지 말고, 꼬리질문을 던져 검증하는 습관을 들인다. (파주)
  3. 비개발자도 자동화 — 코딩을 몰라도 반복 업무 하나 정도는 AI로 자동화(바이브코딩)해, 직접 문제를 해결하는 사람이 된다. (루시·파주)

💡 우리 적용 아이디어 — 교재 지문·강의 스크립트를 넣으면 영상 강의·오디오북·홍보 영상이 자동 생성. Veo(영상)+Lyria(음악·효과음) 결합으로 콘텐츠 제작 시간을 대폭 단축.
🛠 바로 쓸 수 있는 툴 — 영상 Veo·Google Vids + 내레이션 ElevenLabs + 오디오북/요약 NotebookLM (대본은 Gemini).
AI 사진관인공지능 사진관 × Google Cloud
사진을 찍으면 즉시 고객만의 특징을 살린 맞춤형 아바타를 생성하고, 가상 환경의 숏폼 영상으로 즉시 변환해 줍니다. 완성물의 소장과 공유가 직관적으로 연결되어, 오프라인 체험을 소셜 공간으로 확산시키는 강력한 마케팅 툴로 작동합니다.
How it works
  1. App Engine의 프론트엔드·백엔드 분리 아키텍처 위에서 고화질 카메라 입력 스택과 실시간 결합되는 시스템으로 구축.
  2. 원시 이미지 소스는 Gemini 모델 검증을 거쳐 Nano Banana 2로 전송되며, 안면 확장 벡터가 보존된 상태에서 스타일 레이어·시각 스크립트와 합성.
  3. 최종 단계에서 비디오 엔진 Veo 3 프로세스가 병렬 호출되어 프레임 데이터가 지연 없이 인코딩되고, Firestore·Cloud Storage 컴플라이언스 기준에 맞춰 안전하게 저장·통제.
💡 우리 적용 아이디어 — 수강생 수료 인증 아바타·숏폼이나 캐릭터 기반 학습 콘텐츠로 SNS 바이럴 마케팅. 오프라인 이벤트 참여 → 소셜 확산으로 잇는 체험형 프로모션 툴.
🛠 바로 쓸 수 있는 툴 — 이미지 Nano Banana(Gemini) + 아바타·숏폼 HeyGen + 편집 CapCut·Canva.
AI 광고 크리에이티브Nano Banana × Gemini Omni
모델 섭외나 촬영 등의 제약 없이, 원하는 페르소나와 대본 설정만으로 숏폼 리뷰 영상을 즉시 완성합니다. 리소스의 한계를 벗어나 비즈니스 목표와 잠재 고객에 맞춘 다채로운 광고 크리에이티브를 쉽고 빠르게 확장할 수 있습니다.
How it works
  1. Nano Banana 2와 Gemini Omni 아키텍처 결합으로 텍스트·오디오·비디오 레이어가 단일 파이프라인에서 통합 렌더링되는 멀티모달 생성 시스템으로 구축.
  2. 입력된 페르소나와 Gemini 스크립트가 음성 트랙으로 합성되고, 화자의 미세한 움직임과 제품 이미지가 동기화되어 실제 사람이 말하는 듯한 숏폼 비디오로 즉시 렌더링.
  3. 모델 섭외→촬영→편집으로 이어지던 파편화된 물리적 제작 워크플로우가 클라우드 백엔드의 자동화된 단일 스택 아래 통제되어, 리소스 제약이 해결된 솔루션 아키텍처 구현.
💡 우리 적용 아이디어 — 신간·강좌 홍보 숏폼 영상을 모델·촬영 없이 대량 생성. 타깃별(수험생·직장인·학부모) 페르소나와 대본만 바꿔 A/B 마케팅 소재를 빠르게 확장. (← 문의하신 멀티모달 생성 시스템의 핵심 활용처)
🛠 바로 쓸 수 있는 툴 — 대본 Gemini → 영상 Veo·Google Vids → 화자 아바타 HeyGen → 음성 ElevenLabs → 마무리 CapCut. 개발 없이 마케팅 담당 1명이 반나절이면 시제품.
AI 헬스케어AI 피트니스 케어 × Google Cloud
집에서도 실시간 AI 가이드를 통해 부상 위험 없이 전문적인 헬스케어·피트니스 관리를 이어가는 통합 AI 케어 솔루션입니다. 초개인화로 센터 방문 시간·비용을 획기적으로 줄이며, 어디서나 완벽한 건강 관리가 가능합니다.
How it works
  1. 브라우저 환경에서 관절 움직임을 지연 없이 포착하는 MediaPipe 25 아키텍처를 기반으로 구축.
  2. 전반적인 워크플로우는 Cloud Run으로 오케스트레이션되며, 백엔드에서 Gemini·Veo 모델이 연동되어 피트니스 데이터가 동적으로 제어.
  3. 생성된 데이터·미디어 파이프라인은 Firestore·Cloud Storage를 거쳐 엔터프라이즈급 보안 프레임워크에서 안전하게 관리·통제·운영.
💡 우리 적용 아이디어 — 카메라로 동작을 인식하는 기술은 발음 시 입모양 교정이나 유아·체험형 학습의 동작 인터랙션에 응용 가능(교육 적합도는 상대적으로 낮아 탐색 단계).
🛠 바로 쓸 수 있는 툴 — 바로 쓸 완제품은 아직 제한적. 외부 동작인식 학습 솔루션을 검토하는 PoC 단계로.
💬 개발자 없이 어떻게 실현하나요?
위 데모존의 서비스는 Google Cloud·파트너가 직접 만든 맞춤 데모(각각 별개 파이프라인)입니다. 이 파이프라인을 우리가 똑같이 만들 필요는 없습니다. 카드의 아이디어는 ‘직접 구현’이 아니라 ‘어떤 역량을 확보할지’의 방향이며, 개발자 없이 실현하는 현실적 경로는 3가지입니다.
  1. 완제품(SaaS)으로 바로 — 아바타 영상(HeyGen·Synthesia), 영상 생성(Veo·Google Vids), 음성(ElevenLabs), 오디오 요약(NotebookLM) 등 이미 나온 툴을 기획·마케팅팀이 그대로 사용. (개발 0)
  2. 노코드로 가볍게 직접 — Gemini Enterprise·AI Studio로 요약·상담 FAQ·퀴즈 생성 같은 간단한 에이전트를 조립(이번 행사의 ‘루시’·파주 사례처럼 비개발자도 가능). 단, 멀티모달 영상 파이프라인 같은 건 이 범위 밖.
  3. 커스텀은 파트너에 의뢰 — 데모 같은 통합 파이프라인이 꼭 필요하면 SI 파트너(예: 메가존소프트)나 Google Cloud에 구축을 맡긴다(우리가 만들지 않음).
👉 우리 팀(개발자 없음) 권장 순서: ①·② 먼저 파일럿 → 효과 검증 후 필요할 때만 ③.
